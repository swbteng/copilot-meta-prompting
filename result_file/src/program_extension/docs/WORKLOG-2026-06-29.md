## R-EX-11 Fallback UI 완성 + 테스트 모듈 문서화 (코드+테스트+문서)

### 무엇을/왜
스펙(R-EX-11)은 "타임아웃/500 시 무한 로딩 없이 **에러 메시지 + Use original/Cancel Fallback UI**"를
요구하는데, 기존 구현은 (a) 실패를 성공과 구분하지 못했고(`refine()`이 실패에도 원본만 돌려줌), (b)
chat.js 예외 처리는 버튼 없는 텍스트 한 줄뿐이었다. 또 `callRefineApi()`가 HTTP 상태코드를 보지 않아
`text/plain` 500 본문을 정제본으로 오인할 수 있었다. **명세를 완전 구현으로 채우려고** src를 수정했다.

### 변경
1. `src/refiner.js`
   - `callRefineApi()`: 응답 `end`에서 **`statusCode >= 400`이면 `null`**(4xx·5xx를 통신 실패로). text/plain
     500 본문 오인 제거.
   - `refineDetailed(original) -> {text, ok, reason}` 신설(성공/실패 단일 근거): 빈입력=`ok:true(blank)`,
     URL미설정=`ok:true(fixed)`, 서버성공=`ok:true(server)`, 실패=`ok:false(api)`, 예외=`ok:false(exception)`.
   - `refine()`은 `refineDetailed().text`만 돌려주는 **하위호환 래퍼**(기존 계약/테스트 불변). export에 추가.
2. `src/chat.js`
   - `refinePrompt()`: `refineDetailed`가 있으면 그걸로(성공/실패 구분), 없으면 `refine`로(하위호환).
   - `renderFailureFallback()` 신설: `⚠️ ...실패...` 메시지 + **Use original/Cancel** 가로 액션 바(신뢰
     `MarkdownString`, 막히면 `stream.button` 폴백). 핸들러는 `ok:false`/예외에서 이걸 띄우고 자동 전송 안 함.
3. 테스트(`tests/program_extension`)
   - `refiner.test.js`: text/plain 500→원본(상태코드), `refineDetailed` 성공 `ok:true` 케이스 추가.
   - `chat.test.js`: `loadChat`에 `refineDetailed` 주입 지원 + "API 실패 시 에러+Use original/Cancel" 테스트,
     예외 fail-open 테스트를 Fallback 버튼 단언으로 보강. **총 21→24개, 전부 통과.**
   - 테스트 모듈 문서 신설: `tests/program_extension/{AGENTS.md, CLAUDE.md, RUN.md}` + `README.md`/
     `test-results/.../README.md` 갱신(카운트 24, R-EX-11 상세).

### 검증
- `node --check src/refiner.js src/chat.js` OK.
- `npm run test:extension` → **pass 24 / fail 0**. `npm run test:extension:junit`로 `junit.xml`·`summary.txt` 재생성.

### 결정/메모
