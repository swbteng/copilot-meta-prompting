## 정제 서버 라이브 전환 · 구조 단일화 · 문서 최신화 · `${question}` BE 위임

### 배경
- BE `/refine` 경로가 확정됨: `POST https://bgudh8rs8ktr0r-80.proxy.runpod.net/refine`,
  body `{"prompt": "..."}`, 응답 키 `refined`(그 외 `translated_input`/`rewritten_prompt`/`templates` 동반).
- 그동안 `refine()`은 고정 문자열("안녕,클로드")만 반환(서버 연동 전 검증용). 사용자 지시로 **라이브 전환**.
- `program_extension/` 아래에 옛 프록시 시절 `extension/`(v0.6.0) 복사본과 현재 루트(v0.7.0)가 **중복** 존재.
- 문서(README/AGENTS/CLAUDE)에 "고정 문자열 모드가 현재" 등 **옛 상태**가 잔존.

### 한 일
1. **정제 서버 라이브 전환** — `src/refiner.js`
   - `REFINE_API_URL` 기본값을 확정 BE 경로로 설정(환경변수 override 가능), 주석이던 `[정식]` 블록 해제,
     `return FIXED_REPLACEMENT` 제거(URL 미설정 시 폴백으로 강등). `callRefineApi()`는 그대로(응답 첫 키 `refined` 일치).
   - fail-open 보존: 서버 실패/빈 응답이면 원본 반환.
2. **구조 단일화(중복 제거)** — `program_extension/`이 단일 `.vsix` 패키징 루트가 되도록 정리
   - 삭제: `extension/`(옛 v0.6.0 프록시판: engine.js/setup.js/bypass), `refine_copilot.py`, `requirements.txt`(옛 mitmproxy), `run.sh`.
   - `build.sh`·`.vscodeignore`의 "extension/에서 패키징" 잔재 문구 정정. 루트 `src/`가 프록시 잔재를 참조하지 않음 grep 확인.
3. **문서 최신화 + 신규**
   - `RUN.md` 신규(F5 개발/Node 점검/서버 설정/패키징·설치/설정/의존성).
   - `requirements.txt` 재작성: 순수 JS·런타임 의존성 0이라 **의존성 없음**을 주석으로 명시(패키징만 `npx @vscode/vsce`).
   - README/AGENTS/CLAUDE에서 "고정 문자열 모드"/"서버로 전환해야"/"주석된 서버" 등 옛 서술을 서버 모드 기준으로 정정.
4. **`${question}` 자리표시자 — BE 위임으로 결정** (이 세션 마지막)
   - 조사: 정제본은 `${question}`(응답마다 `${query}` 등으로 이름이 바뀜)을 **안 채운 채** 옴. 모델이 직전
     `@refine` 대화 맥락 덕에 우연히 답했던 것 → 취약. 한때 확장에서 `appendQuestionDefinition()`으로 말미에
     `${var} = <원본질문>` 정의를 덧붙였음(치환 아님).
   - **결정 변경**: 자리표시자 채움은 **BE 서버가 처리**. 확장 측 `appendQuestionDefinition`은 함수·호출·export를
     **전부 주석/보류**로 남김(되살리는 법 주석에 명시). 실측상 BE가 이미 "사용자 입력: …"을 채워 보냄을 확인.
5. **워크로그 필수화** — AGENTS §7(Definition of Done)에 `docs/WORKLOG-<날짜>.md` 작성을 **생략 불가 항목**으로
   추가, CLAUDE.md 요약에도 명시. (이 파일이 그 첫 적용)

### 배경

- `@refine` 미리보기의 승인 UI가 `stream.button` **5개 세로 스택**이라 길고 투박했다. 가로·현대적 배치 요청
  (LG 디자인 참고).

### 한 일
1. **`.env` 정리 + ignore**
   - `.env` 신규(현재 쓰는 `REFINE_API_URL` 1개 명시) + `.env.example` 템플릿 신규.
   - `.gitignore`에 `.env`/`.env.local`/`.env.*.local` 추가(`.env.example`만 추적). `.vscodeignore`에
     `.env`/`.env.example` 추가(.vsix 제외).
   - 코드 내장 기본값은 그대로 둠 → `.env` 없어도 동작. `.env`는 Node 점검/패키징 쉘 전용(확장 런타임/F5는
     자동 로드 안 함)임을 `.env`·`RUN.md`에 명시. **런타임 의존성 0 유지**(dotenv 등 미도입).

   - 세로 `stream.button` 5개 → **codicon 커맨드 링크 가로 1줄**(신뢰 `MarkdownString`,

   - 명령/인자/로깅 동작은 그대로(allow는 `[원본,정제본]`). 커맨드 링크가 막힌 환경이면
     `renderActionsFallback()`로 **기존 세로 버튼 폴백**(fail-open).

     색은 채팅 표면에 적용 불가. 그래서 LG 디자인은 *원칙*(1차 CTA 강조·간결 라벨·컴팩트 가로 배치)으로만 반영.
3. **문서 정합화**: README(빠른 시작/사용법/정제 동작 + 스테일했던 `extension/`→`program_extension/` 폴더명),
   AGENTS(§0 요약·§2 코드맵/액션 표 제목), RUN.md(버튼→가로 액션 바, 스테일 `${question}` 정의 문구 제거,
