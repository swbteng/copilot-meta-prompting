### 배경 (평가 기준 점검 중 발견)
평가 기준(6.테스트 / 4.RUN.md DevOps 8개) 충족 여부를 점검하다 **결정적 결함**을 발견:
- `src/refiner.js`가 **로드 자체 불가** — 파일 상단 블록(헤더 주석, `bakedRefineUrl()` 함수 +
  `const REFINE_API_URL`, 보류 주석 블록)이 **통째로 중복 붙여넣기**되어 `const REFINE_API_URL`이
  2회 선언 → `SyntaxError: Identifier 'REFINE_API_URL' has already been declared`. (06-29 작업의 편집 잔재로 추정)
- 그 결과 현재 코드로 테스트 재실행 시 **19 pass / 6 fail**(`refiner.test.js` 6건이 실코드 로드 단계에서
  실패). 그러나 첨부된 `test-results/program_extension/{junit.xml,summary.txt}`는 **25 pass / 0 fail**로
  남아 있어 "코드↔테스트↔결과" 매칭이 깨진 stale 상태였음.
- 런타임 영향: `extension.js → chat.js → require('./refiner')`도 같은 SyntaxError로 깨짐. **모듈 로드
  단계 에러라 fail-open(try/catch)으로도 못 잡음** → `@refine`가 기동/첫 호출에서 깨질 위험(R-EX-11 무력화).
