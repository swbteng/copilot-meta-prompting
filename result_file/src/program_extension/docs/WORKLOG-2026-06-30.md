## (추가) 테스트 경로를 소스 구조와 일치(미러링)

### 배경
표준 관례는 `tests/`를 `src/`와 같은 구조로 두는 것(예: `src/a/b.js → tests/a/b.test.js`). 그런데 확장 코드는
`src/program_extension/**src**/*.js`에 있는 반면 테스트는 `tests/program_extension/*.test.js`로 **안쪽 `src/`
레벨이 빠져** 경로가 한 칸 어긋나 있었다(채점의 "코드↔테스트" 미러 신호가 약함).

### 한 일
1. **테스트 파일 이동** — 5개 `*.test.js`를 `tests/program_extension/` → **`tests/program_extension/src/`**로 이동
   (소스 `src/program_extension/src/`의 정확한 미러). 공통 인프라 `_helpers/`(언더스코어 = 소스 대응 없음)는
   모듈 테스트 루트에 유지.
2. **상대경로 보정** — 한 단계 깊어졌으므로 각 테스트의 실코드 로드 경로 `__dirname,"..","..",...` →

3. **스크립트 glob 갱신** — 루트 `package.json`의 `test:extension`·`test:extension:junit` 글롭

4. **문서 갱신** — `tests/program_extension/{AGENTS.md,README.md,RUN.md}`의 글롭/단일파일 경로/구조 설명을

## (추가) build.sh 의존성 0 빌드로 재작성 (Node/vsce 제거)

### 배경
기존 `build.sh`는 **Node + npx + `@vscode/vsce` + 네트워크**가 있어야 빌드됐다. "그런 게 전혀 없는 컴퓨터"에서도
`./build.sh`만으로 `.vsix`가 나오게 해달라는 요청. `.vsix`는 결국 **OPC 규격 ZIP**(`extension.vsixmanifest`
+ `[Content_Types].xml` + `extension/<파일들>`)이라, **Windows 기본 내장 PowerShell**만으로 직접 만들 수 있다.

### 한 일
1. **vsce 제거 → PowerShell 직접 패키징** — `build.sh`가 `npx @vscode/vsce` 대신, 임시 `.ps1`을 생성해
   `powershell.exe`로 실행. PowerShell이 (a) `package.json`을 `ConvertFrom-Json`으로 읽어 매니페스트 메타데이터
   추출, (b) `extension.vsixmanifest`·`[Content_Types].xml` 생성(BOM 없는 UTF-8), (c) `System.IO.Compression`으로
   ZIP 압축(엔트리 경로는 forward-slash 강제). 매니페스트 구조는 vsce 출력을 그대로 재현.
2. **`.env` 굽기를 Node → 순수 셸로** — `grep`/`sed`로 `REFINE_API_URL`을 뽑아 `src/env.generated.js` 생성
   (CRLF·따옴표 처리 + JS 문자열 이스케이프). Node 호출 제거.
3. **메타데이터를 package.json에서 동적 추출** — 이름/버전/퍼블리셔/설명/엔진/카테고리 하드코딩 없음.
   (사용자가 바꾼 `teamPara.copilot-meta-prompting-refiner@0.8.0`도 그대로 반영됨을 확인.)

### 발견·수정한 버그
- **PowerShell `-Include` + `-LiteralPath` 함정**: `Get-ChildItem -LiteralPath $d -Recurse -File -Include '*.map',...`
  는 `-Include`가 **무시되어 전체 파일을 반환** → `Remove-Item`이 `extension/`을 통째로 비웠고, 결과 `.vsix`에
  루트 2개 파일만 들어갔다. `Where-Object { $_.Name -like ... }`로 교체해 해결.

### 검증
- `./build.sh` → `copilot-meta-prompting-refiner-0.8.0.vsix` 생성, 내부 **14 엔트리**
  (`extension.vsixmanifest`, `[Content_Types].xml`, `extension/package.json`+`readme.md`+`src/`7개+`media/`3개).
  forward-slash 확인, backslash 0. 한글 `Description`이 `package.json`과 **바이트 단위 일치**(인코딩 정상).
- **실제 설치 성공**: `code --install-extension ...` → "successfully installed", `code --list-extensions`에
  `teampara.copilot-meta-prompting-refiner@0.8.0` 노출. 검증 후 **언인스톨 + 산출물 .vsix 제거**(기존
  `swbc.swbc-prompt-refiner@0.7.0`만 남김 — id 중복 충돌 방지).
- 전제: 이 스크립트를 돌릴 셸(git bash) + Windows 기본 PowerShell만 있으면 됨. Node/npm/npx/vsce/네트워크 불필요.

### 문서
- `RUN.md`(§4 패키징·§6 의존성), `requirements.txt`, `AGENTS.md`(머리말·§5)를 "패키징 의존성 0(PowerShell만)"로 갱신.

## (추가) build.sh 를 '하이브리드'로 전환 (vsce 선호 + PowerShell 폴백)

### 배경 / 결정
사용자가 목표를 명확히 함: *"의존성 없는 상태를 **유지**"가 아니라 "의존성 없는 상태에서도 빌드가 **가능**"*.
→ 평소엔 **표준 도구(@vscode/vsce)**를 쓰되, 아무것도 없는 머신에서도 빌드되게 한다. 가장 충실한 형태는
**하이브리드**(표준 선호 + 무의존 폴백). "manifest 선언 후 설치 뒤 빌드"는 좋은 원칙이지만, 그 manifest는
`requirements.txt`(pip)가 아니라 **`package.json`의 devDependencies**가 맞다(이 프로젝트는 JS 확장).
폴백을 'Node 자동설치'로 두지 않은 이유: npm=Node 동봉이라 'npm 설치'='Node 설치'이고 늘 네트워크+머신변경이
필요. PowerShell 폴백이 오프라인·무변경이라 그 케이스를 더 잘 커버(중복 회피).

### 한 일
1. **`package.json`** — `devDependencies: { "@vscode/vsce": "3.9.2" }` 추가(버전 핀). `package` 스크립트도
   `vsce package --allow-missing-repository`로.
2. **`build.sh` 하이브리드화** — 함수 분리: `bake_env`(공통, 셸) / `package_with_vsce`(Tier 1) / `package_with_powershell`(Tier 2).
   - Tier 1: `node`/`npm` 있으면 `npm install`(devDeps 핀) → `npx @vscode/vsce package`. 실패하면 비0 반환.
   - Tier 2: vsce 불가/실패 시 직전에 만든 PowerShell 직접 패키징 경로(설치·네트워크 0).
   - 디스패치: `bake_env; if ! package_with_vsce; then package_with_powershell; fi`.

### 검증 (두 경로 모두)
- **Tier 1(vsce)**: `./build.sh` → vsce가 `copilot-meta-prompting-refiner-0.8.0.vsix`(14파일) 생성. 정상.
- **Tier 2(PowerShell)**: 디스패치를 강제 폴백으로 바꾼 임시 복사본 실행 → 동일 14엔트리, forward-slash,
  한글 Description 바이트 일치, Identity(id/ver/pub) 정상. 두 경로 결과물 동등.
- `node_modules`(@vscode/vsce@3.9.2)는 `.gitignore`/`.vscodeignore`로 추적·패키징 제외 → `.vsix`에 안 들어감.

### 문서
- `RUN.md`(§4·§6), `AGENTS.md`(머리말·§5), `requirements.txt`를 '하이브리드(Tier1 vsce / Tier2 PowerShell 폴백)'로 갱신.
