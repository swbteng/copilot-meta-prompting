# WORKLOG 2026-06-25

## P6 구현 — `@refine` Chat Participant (확인형 정제) + 프록시 bypass

### 배경
- 사용자가 `./build.sh --install` 후 Copilot Chat에서 `@refine`가 안 잡힌다고 보고.
- 원인: P6는 그동안 **문서(계획)로만** 존재. `package.json`에 `contributes.chatParticipants` 없고
  코드에 `createChatParticipant` 없음 -> VS Code가 참가자를 등록할 수 없었음. 그래서 구현 착수.
- 슬래시 컨벤션: 직전 합의(bare `@refine` 기본 + 단일 `/bypass`)에서 사용자가 **"modify만 `/bypass`"** 최종 승인.
  -> allow/use-original은 확장이 마커를 직접 부착해 슬래시 없이 전송, **슬래시가 노출되는 곳은 modify 시드 하나뿐**.

### 한 일
1. **프록시(`refine_copilot.py`) bypass 마커**
   - CONFIG에 `BYPASS_MARKER`(zero-width 5자, `U+200B U+2063 U+2060 U+2063 U+200B`) 추가(`REFINE_BYPASS_MARKER` 환경변수로 교체 가능).
   - 순수 함수 `strip_bypass_marker(text) -> (cleaned, bool)` 추가. `refine_text()` 맨 앞에서 호출 ->
     마커 있으면 **정제 없이 마커만 제거해 반환**. `refine_text()`가 messages[]·input[]·WS 모든 경로의 refiner라 한 곳만 고쳐 전 경로 커버.
   - `load()` 로그에 bypass 활성 1줄 추가. **절대 제약 5개 불변**(마커=통과는 allowlist 연장, fail-open 유지).
2. **확장 신규 파일**
   - `extension/src/refiner.js`: JS 정제(`refine`, 현재 고정 문자열 "안녕,클로드") + `attachBypass()` + `BYPASS_MARKER`
     (프록시와 동일 코드포인트) + 서버모드용 `callRefineApi`(http/https, 미사용/주석 경로).
   - `extension/src/chat.js`: `@refine` 참가자 등록 + 미리보기(`stream.markdown`) + 5버튼(`stream.button`) +
     5개 명령. 버튼 동작:
     - **allow / use original** -> `sendBypassed()`: `attachBypass()`로 마커 부착 후 `workbench.action.chat.open({query})` 자동 전송.
     - **modify** -> `seedInput("@refine /bypass <정제본>")`(`isPartialQuery:true`) -> 유저 편집 -> 엔터 시 `/bypass` 핸들러.
     - **cancel** -> `seedInput("@refine <원본>")` -> 엔터 시 기본 흐름 재진입(재정제).
     - **try again** -> `@refine <원본>` 자동 전송 -> 새 미리보기 턴.
   - `extension/src/extension.js`: `registerChat(context)` 호출 추가(chat API 없는 구버전이면 내부에서 조용히 비활성).
3. **`extension/package.json`**: `contributes.chatParticipants`(id `swbc.refine`, name `refine`, command `bypass`) 추가,
   `engines.vscode` `^1.85.0` -> `^1.90.0`(chat API 정식), 버전 `0.5.0` -> `0.6.0`.
4. **빌드/설치**: `./build.sh --install` -> `swbc-prompt-refiner-0.6.0.vsix`(engine/refine_copilot.py 동봉 갱신 포함) 설치 완료.

### 검증
- `_selftest_bypass.py`(작성->실행->삭제): strip/refine_text/messages[]/input[] 경로에서 마커 부착 시 정제 건너뛰고 마커 제거,
  마커 없으면 평소대로 고정 문자열 교체 — **10/10 PASS**.
- JS/PY `BYPASS_MARKER` 코드포인트 일치 확인(`200b 2063 2060 2063 200b`).
- `node --check` refiner.js / chat.js / extension.js 구문 통과.

### 결정
- "modify만 `/bypass`" 확정: allow/use-original은 마커 직접 부착(슬래시·마커 비노출), modify만 입력창에 `/bypass` 시드.
- 마커는 zero-width(보이지 않음) 우선. **fail-open**이라 마커가 정규화로 사라져도 최악은 '이중 정제'(Copilot 정상 동작).

### 남은 일 (라이브 손 확인 — Extension Development Host / 설치본)
1. `@refine`가 Copilot Chat 참가자로 뜨고 미리보기+5버튼 렌더되는지.
2. `workbench.action.chat.open`의 자동 전송(allow) vs `isPartialQuery:true` 입력창만 채우기(modify/cancel)가
   설치된 VS Code 버전에서 기대대로인지.
3. zero-width 마커가 입력창->요청 본문까지 생존하는지. 사라지면 `REFINE_BYPASS_MARKER`(프록시) + `src/refiner.js` 마커를
   **가시 토큰**으로 함께 교체.
- (기존 공통) 실제 공유 전: `refine_text()` 고정 문자열 -> 서버 모드 전환은 사용자 지시 + 서버 스키마 확정 후.
